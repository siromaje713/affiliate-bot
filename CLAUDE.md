## 黄金パターン

1. 知識暴露型：「[事実]って[根拠]。[返信誘導]？😳」
2. 行動訂正型：「[NG行動]してる人まだいる？[正解]が正解。やってた？」
3. やり方暴露型：「[方法]って[意外な事実]知ってた？[返信誘導]🙌」
4. 保存型リスト：「[テーマ]【保存用】悩み→解決策×5〜8項目」
5. 姉シリーズ：「姉が〜」で始まるリアル体験談風（G-1〜G-10）

## 品質基準（Output Gate）

- フック（1行目）で手が止まるか？
- 140文字以内で価値が伝わるか？
- CTAが自然か？
- スコア70点未満→差し戻し再生成

## データフロー

docs/raw/ → 素材（research_YYYYMMDD.json）
docs/wiki/ → 構造化知識（パターン別・トピック別）
docs/pipeline.csv → 投稿振り分け（planned→draft→review→scheduled→published）
docs/outputs.csv → 投稿実績+パフォーマンス
winning_patterns.json → バズパターンDB（実績スコアで自動更新）

## 判断履歴（やらかしDB）

- imgur CDN → shadowban。テキストonly or Fal AI直URL
- Threads API sinceパラメータ → 400エラー。使うな
- GitHub UIでファイル直接編集 → 破損。Claude Code経由のみ
- アフィリンク直貼り → エンゲージ激減。アカウントパワー優先
- like_count他人投稿 → API非対応。ベンチマークは手動確認値のみ

## 絶対ルール

- APIキー/トークンをCLAUDE.md・ログ・コードに書くな
- テストなしでデプロイするな（最低pytest）
- git push --force / rm -rf 禁止
- Python 3.10+構文禁止（Renderが3.9）
- 変更後 python3 -c "import orchestrator" で構文チェック
- 関係ないファイルを「ついでに」変更するな
- 修正後は必ず動作確認（Slackログ or dry-run）してからセッション終了
- pushしてないコミットを残してセッション終了するな
